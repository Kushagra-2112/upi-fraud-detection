## 🧠 How It Works

Traditional fraud detection uses simple rules like transaction limits. FraudShield analyzes **behavioral patterns** instead:

```
✅ Normal Transaction:
   Sender ₹9L → ₹0   +   Receiver ₹5K → ₹9.05L   (money arrived)

🚨 Fraud Transaction:
   Sender ₹9L → ₹0   +   Receiver ₹5K → ₹5K       (money vanished!)
```

> The model asks — *"money left the sender, but where did it go?"*

**Random Forest** is the deployed model — best balance of precision and recall with only **29 missed fraud cases** out of 1,643 in the test set.

## 📌 Key Findings

- 🔑 Fraud is detected by **behavioral patterns**, not transaction amount alone
- 🥇 Top fraud signal: `balance_diff` — how much money disappeared from sender
- 🚨 `dest_balance_unchanged` — receiver balance not changing is a strong fraud signal
- 🌙 `is_odd_hour` has almost **no importance** — fraudsters don't only operate at night
- ⚖️ SMOTE was critical — raw data had only **0.129% fraud** (8,213 out of 6,362,620 transactions)

## 🚀 Local Setup

```bash
# 1. Clone the repo
git clone https://github.com/Kushagra-2112/upi-fraud-detection
cd upi-fraud-detection

# 2. Create and activate virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
source venv/bin/activate       # Mac/Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Download PaySim dataset from Kaggle
# 👉 https://www.kaggle.com/datasets/ealaxi/paysim1
# Place the CSV inside the data/ folder

# 5. Run the training notebook
# Open notebooks/01_eda_and_training.ipynb and run all cells
# This generates the .pkl model files inside models/

# 6. Launch the app
streamlit run app.py
```
